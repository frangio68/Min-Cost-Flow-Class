Detailed Description

These macros control some important details of the implementation.

Although using macros for activating features of the implementation is not very C++, switching off some unused features may make the code more efficient in running time or memory.

Macro Definition Documentation

◆ DYNMC_MCF_ZIB

#define DYNMC_MCF_ZIB   0

Decides if the graph topology (arcs, nodes) can be changed.

If DYNMC_MCF_ZIB > 0, some of the methods of the public interface of class that allow to change the topology of the underlying network are actually implemented. Possible values of this macro are:

  • 0 => arcs cannot be added, deleted or changed start/end node, all the other operations are possible;
  • 1 => all the methods that change the topology of the graph are implemented.

◆ SWAP_BASKETS

#define SWAP_BASKETS   1

Decide one important details of the pricing implementation.

In the simplex algorithm for MCF, one of the critical points is how "pricing" is done, i.e., how the next entering/leaving variables is selected to be inserted to/removed from the current base. This class has several different pricing strategies, some of which being based on keeping lists of "candidate" arcs into some data structures organized in "baskets", that are ordered according some criteria. When vectors of baskets are ordered, it is possible either to order the list itself, thus swapping baskets (which are structures of some size), or to order a corresponding vector of pointers which points to the (unordered) original vector of baskets. If SWAP_BASKETS > 0 the first approach is taken, while if SWAP_BASKETS == 0 the second one is used, In generale, swapping pointers is faster, but there may be cases where phisically swapping the baskets is preferable.
